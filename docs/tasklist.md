## Phase E: LLM実働化（Phase C 安定後）

LLMは新システム（個別株バスケット + PTS）のベースライン性能が確定してから導入する。
理由: (1) ETF版で蓄積したLLM評価データは個別株版に移行した時点で無意味になる
(2) 執行改革とLLMを同時に入れると効果の帰属が分離できない
(3) +3-5%の改善は執行改革（+10-15%）の後で載せればよい

- [ ] **E-1**: LLM実働化 — 環境変数を設定し Phase p90_plus_llm で稼働開始
  - `LLM_PROVIDER=openrouter`, `OPENROUTER_API_KEY`, `PHASE=p90_plus_llm`
  - 既存実装で動作するため開発不要
  - **前提**: Phase C のペーパートレード（C-5）で新システムのベースラインが確定していること
- [ ] **E-2**: LLM判定品質の検証（3ヶ月）
  - passAlpha > 15bps、excludeAlpha < 10bps、差 > 5bps を月次で確認
  - 基準未達が2ヶ月連続なら LLM を無効化し P90 のみに切り替え
  - counterfactual return記録（実装済み）で除外日のリターンを追跡
