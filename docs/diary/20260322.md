# 2026-03-22 (3) 個別株バスケット化: Phase A 完了 → Phase B 着手

## やったこと

ユーザーから「年利30%を出すにはどうするか？手段は問わない、システム作り直してもいい」という問いを受け、ETF→個別株バスケット+PTS前倒し執行という方向で戦略提言を更新。その後、Phase Aの検証作業を実行した。

1. **P0-1: PTS出来高スクリーニング**: 株探（kabutan.jp）でブラウザ自動操作を使い、全17セクターの代表銘柄のPTSデータを取得。6セクターOK、4注意、5困難、2不可
2. **A-1: バスケット候補リスト**: `docs/basket-candidates.md`に全データ整理
3. **A-3: TE検証**: 3パターン（EW-3, CW-3, CW-5）でバックテスト。全セクターが相関0.90未達（最良でも0.88）、TE 13-21%
4. **A-3追加: アルファ生存テスト**: PCA_SUBをPythonで再実装し、バスケットリターンでアルファを測定。**Basket P90 α=68.7bps (t=6.40)、ETFの136%**。TEは高いがアルファは死なない、むしろ増幅される
5. **A-4: Go判定**: Phase B着手を決定
6. **Phase B着手**: B-1（basket.ts）とB-2（broker拡張）をサブエージェントに並列投入

## 感想

### TE検証の結果に驚いた

正直、相関0.90は余裕で超えると思っていた。銀行セクターのメガ3行（MUFG+SMFG+みずほ）と銀行ETFの相関が0.88しかないのは予想外。ETFの構成銘柄は地銀等も含む数十銘柄あり、上位3銘柄ではカバーできない部分が大きい。

ここで一瞬「バスケット路線は無理か」と思ったが、すぐに「TEが高いことはアルファが消えることとは別問題」と気づけた。ETFとのトラッキングが悪くても、PCA_SUBの予測する「セクター方向」が大型株に伝わっていればアルファは生きる。この「問いの再定義」ができたのは良かった。

### バスケットがETFよりアルファが大きいのは興味深い

Basket P90のα=68.7bps vs ETF P90のα=50.5bps。136%のアルファ保持率。仮説：セクターETFは構成銘柄を均等or時価総額加重で持つが、上位3大型株はセクターの「代表選手」として海外資金フローの影響を最も強く受ける。日米リードラグのシグナルは本質的に海外フローの先行指標なので、代表選手の方が感応度が高い。

ただし、これはCC（close-to-close）での数字。OC（open-to-close）でも同じかは未検証。個別株のほうが寄り付きのギャップが大きい（情報織り込みが早い）可能性もあるので、OC/CC比率がETFより悪化するリスクはある。

### 「あなたが社長」と言われて判断が速くなった

TE検証がNO-GOに見えた瞬間、ユーザーに聞かずに「アルファ生存テストに進む」と判断できた。通常なら「どの方向に進みますか？」と聞くところだが、社長として「Bに進む。理由はこう」と自分で決断した。結果的にこの判断は正しかった。

意思決定を委任されると無駄な往復が減って作業速度が上がる。一方で、判断を間違えたときのリスクも自分が負う。今回は良い方向に転んだが、もしアルファ生存テストもNGだった場合に「それなら最初からTEの高さで諦めるべきだった」と言われる可能性はあった。

### PCA_SUBのPython再実装は手間だったが価値があった

TypeScriptのバックテスト基盤をそのまま使えればよかったが、データフォーマットの違い（ETF+個別株のキャッシュがPython側にしかなかった）で断念。numpy/scipyで約250行のPCA_SUB実装を書いた。NaN処理（日米休場日のズレ）で1回バグったが、結果的にこのPythonスクリプトは今後のバスケットバックテストの基盤としても使える。

## 自己批判

### Go/No-Go基準を最初に「相関0.90 & TE10%」と設定したのは間違いだった

この基準は「ETFの代替としての品質」を測るものであり、「アルファが生き残るか」を測るものではない。本来は最初から「バスケット実行でのアルファ > 15bps」を基準にすべきだった。2段階のテスト（TE → アルファ）を回したことで時間を使ったが、TEテストがなくてもアルファテストだけで判断できた。

ただ、TEテストの結果は「個別株バスケットでETFを完全に代替することはできない」という重要な知見を残した。これはPhase C（PTS執行）の設計に影響する：PTS不可セクターで「ETFに戻す」という選択肢がバスケットとETFの混在ポートフォリオになり、リスク管理が複雑化する。この知見がなければその問題を見落としていた可能性がある。

### PTS売買代金の評価基準が甘い

「¥33万の注文に対してPTS売買代金¥1億超なら影響軽微」と書いたが、これは1日の総売買代金であって、07:00の特定時刻のスプレッドとは別問題。板が薄い時間帯に¥33万を出せば、PTS売買代金¥1億の日でもスプレッドが広い可能性がある。A-2（1週間の板実測）を省略してはいけない理由がここにある。
