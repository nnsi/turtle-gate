# 戦略提言: 個別株バスケット化による執行改革

日付: 2026-03-22

---

## 1. 現状の診断

### 1.1 シグナルは生きている

PCA_SUBのアルファは統計的に有意:

- Day 1 CC alpha: 21.9bps (t=10.06)
- OOS検証（2020-2025）: P90 α=29.0bps (t=4.03)
- 五分位モノトニシティ: Q5=22.5bps vs Q1=-0.6bps（ただし2024-2025で劣化兆候あり）

バックテスト上のCC gross ARはP75で28.5%、P90で17.0%。シグナル自体には十分なアルファが存在する。

### 1.2 殺しているのはETF

TOPIX-17 ETFの構造的制約がアルファを毀損している:

| 問題 | 数値 | 影響 |
|------|------|------|
| OC/CC比率 | 0.40-0.46 | アルファの54-60%が寄り付きで消失 |
| 日次出来高 | ¥1,100万〜¥6,700万 | ¥100万の注文で板の数%を食う |
| PTS流動性 | **ゼロ**（取引実績なし） | 前倒し執行が不可能 |
| 片道スプレッド | 5.2-10.8bps（平均7.36bps） | BEマージンを圧迫 |
| キャパシティ | ¥1,000万が実質上限 | 事業として成立しない |

出典: 株探 2026-03-19（出来高）、JPX月次データ（スプレッド）、株探PTSランキング（PTS流動性）

### 1.3 結論

**シグナルを捨てるのではなく、器を変える。**

## 2. 提案: TOPIX-17 ETF → 個別株バスケット

### 2.1 概要

PCA_SUBが「銀行セクターをロング」と出力したとき、ETF(1631.T)を買う代わりに、**三菱UFJ(8306)、三井住友(8316)、みずほ(8411)を均等で買う**。

### 2.2 個別株バスケットで解決する問題

| 問題 | TOPIX-17 ETF | 個別株バスケット |
|------|-------------|----------------|
| 東証日次出来高 | ¥1,100万〜¥6,700万 | 大型株単体で¥100億〜¥500億超 |
| PTS流動性 | ゼロ | 主要銘柄はPTS出来高あり |
| 片道スプレッド | 5-11bps | 大型株は1-3bps |
| マーケットインパクト | ¥100万で板の数% | ¥1億でも誤差 |
| キャパシティ | ¥1,000万上限 | ¥10億でも可能 |

### 2.3 PTS前倒し執行の復活

TOPIX-17 ETFではPTS取引実績ゼロだったが、主要個別株（トヨタ、ソニー、三菱UFJ等）はPTSで実際に取引されている。

個別株バスケットに切り替えることで:

```
06:00 シグナル生成（PCA_SUB → セクター候補）
06:05 セクター候補 → 個別株バスケットに変換
07:00 PTS で仕込み（前日終値に近い価格で約定）
15:00 東証引けで手仕舞い
```

OC/CC比率を0.45→0.65-0.70に改善できる可能性がある。これがリターン改善の最大のレバー。

### 2.4 トラッキングエラー

セクターETFと個別株バスケットの乖離はリスクだが、制御可能:

- 各セクターの時価総額上位3-5銘柄で構成すれば、ETFとの相関は0.95+が期待できる
- バックテストでETFリターンと個別株バスケットリターンの乖離を定量検証可能
- セクター内の銘柄選別でアルファ上乗せの余地がある（上振れ要因）
- 要求定義書§4.2で「将来拡張候補」として明記済み

### 2.5 レバレッジ

個別株の信用取引で2-3倍。確信度に応じたレバレッジ:

| バンド | レバレッジ |
|--------|-----------|
| P90超 | 2.0倍 |
| P80-P89（LLM tailwind） | 1.5倍 |
| P80-P89（LLM neutral） | 1.0倍 |

P90 OC gross 7.8% × 2倍 = 15.6%。個別株PTS執行でOC/CC改善後ならさらに上。MDD悪化は避けられないが、P90（MDD -3.7%）の2倍でもMDD -7〜-8%程度。運用可能な範囲。

| 施策 | 効果 | 前提条件 |

## 6. システム実装ロードマップ

### Phase A: 検証（1-2週間）

個別株バスケットの実現可能性を確認する。コードは書かない。

1. **主要個別株のPTS出来高確認**: 17セクター × 上位3-5銘柄の東証/PTS出来高を株探で調査
2. **セクター代表銘柄リスト作成**: 時価総額上位で流動性が十分な銘柄を各セクター3-5銘柄選定
3. **バックテスト検証**: ETFリターンと個別株バスケットリターンの相関・乖離を過去データで計測
4. **トラッキングエラー許容範囲の決定**: 相関0.95未満のセクターを除外するか、銘柄数を増やすか

### Phase B: バスケット変換モジュール実装（1-2週間）

Phase Aの結果がGo判定の場合:

1. **`src/basket.ts`新規**: セクターシグナル → 個別株バスケットへの変換ロジック
   - セクター→銘柄マッピング定義
   - 時価総額加重 or 均等加重の選択
   - 銘柄別の流動性フィルター
2. **`src/broker-sbi.ts`新規（or `broker-kabu.ts`拡張）**: 個別株の発注対応
3. **`execute.ts`改修**: バスケット注文への対応（1セクター1注文 → 1セクターN銘柄注文）
4. **`unwind.ts`改修**: 個別株ポジションの一括決済

### Phase C: PTS執行（2-3週間）

Phase Bの個別株バスケットが安定動作した後:

1. **PTS板の実測記録**: 代表銘柄のPTSモーニング（08:20-09:00）板情報を1週間記録
2. **`broker-pts.ts`新規**: PTS注文アダプタ
3. **`execute.ts`の2段階化**: PTS先行 → 東証補完

### Phase D: レバレッジ（Phase B安定後）

1. **信用取引対応**: `execute.ts`のポジションサイズ計算にレバレッジ倍率を追加
2. **リスク管理強化**: レバレッジ時のMDD上限チェック、日次損失制限

## 7. リスクと留保事項

### 7.1 検証前の不確実性

- **個別株バスケットのトラッキングエラー**: セクターETFとの乖離が大きければ、PCA_SUBシグナルの有効性が低下する。バックテストで要検証
- **個別株のPTS流動性**: 主要銘柄でもPTSモーニング時間帯の板が薄い可能性。実測が必要
- **執行複雑性の増大**: 1セクター1注文 → 1セクター3-5注文。約定管理、部分約定処理が複雑化
- **信用取引のコスト**: 金利（年2-3%程度）がレバレッジ分のリターンを削る

### 7.2 2024-2025のモノトニシティ崩壊

直近サブ期間（2024-2025）でQ5=12.5bps vs Q1=4.6bps（Q2=-7.4bps）と五分位の単調性が崩れている。アルファの構造変化の兆候である可能性があり、個別株バスケット化で解決する問題ではない。Cfullドリフトモニター（実装済み）での早期検知が必須。

### 7.3 30%は保証ではない

合成シナリオの34%は最も楽観的な見積もり。全ての前提（PTS流動性、トラッキングエラー、LLM品質、レバレッジ許容度）が好転した場合の数字。保守的には15%、中央値で20-25%がより現実的。
