# Conviction Filter Threshold Scan (P60–P90)

PCA_SUB戦略のConviction Filterにおける最適パーセンタイル閾値を探索した結果。
データ: `data/closes.csv` (1779日 x 28銘柄), パラメータ: L=60, K=3, λ=0.9, q=0.3

## 2. 主要な発見

### Trade-day平均αは閾値に依存しない

全閾値でtrade-day平均αは31〜38 bpsの狭い範囲に収まる。Conviction Filterは「上位を厳選する」機能ではなく、「明らかにダメな日を弾く」機能であることを示唆。

### BEは閾値が高いほどやや有利

閾値を上げると取引頻度が下がり、平均ターンオーバーが減少するためBEが微増する（P60: 16.0 → P90: 18.8 bps）。ただし差は小さい。

### リターンとMDDはトレードオフ

閾値を下げる → 取引機会増 → ARが増えるが、MDDも悪化する。

## 3. MDD制約別の推奨閾値

| MDD許容水準 | Gross基準での推奨 | Net @8bps基準での推奨 |
|------------|------------------|---------------------|
| -20%       | P90のみ           | 該当なし（P90でも-22.1%） |
| -25%       | P75              | P90のみ（-22.1%）    |
| -30%       | P60〜P70          | P75（-29.0%）       |

## 4. OOS（アウトオブサンプル）信頼性

IS期間(2015-2019)で算出した閾値をOOS期間(2020-2025)に適用した結果：

- 全閾値でOOS trade-day αはプラス（31〜35 bps）
- t統計量はP60で6.92、P90で2.60。閾値を下げるほどサンプル数が増えt値が改善
- OOS strategy AR(gross)はP60: 30.4% → P90: 8.8%

## 5. 結論（暫定）

純粋なリスクリターン最適化ではP60〜P75が優位。ただしMDD制約が厳しい場合はP80〜P90が必要。運用フェーズに応じた使い分けが現実的：

- **小額検証フェーズ**: P75（Net R/R 1.25, MDD -29%@8bps）
- **本格運用フェーズ**: P80〜P90（MDD -22〜-26%@8bps）
